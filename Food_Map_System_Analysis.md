# Alma 美食地圖 - 系統分析與需求分析文件

## 1. 系統概述
本系統旨在將 `Alma_Food_Map_Final.xlsx` 中的美食資料轉化為一個可交互的「美食地圖」Web App。設計風格將完全參照「全國廁所快找 (TOILETS_System_Analysis.md)」之極簡、高對比、極速響應之 PWA (Progressive Web App) 架構，提供使用者在行動裝置上快速搜尋周邊美食的能力。

## 2. 實作可行性評估
- **技術可行性 (高)**: 採用成熟的 Leaflet.js 地圖引擎與靜態 JSON 架構，可確保低成本部署與高效能運作。
- **資料可行性 (中)**: 原始資料為 Excel 格式，包含店名、地址、電話、營業時間與食記連結。需額外進行一次性的「地址轉經緯度 (Geocoding)」處理，以利地圖標記。
- **離線支援 (高)**: 透過 Service Worker 技術，可達成 100% 離線檢索與地圖存取。

## 3. 軟體架構 (Software Architecture)
- **前端核心**: 原生 HTML5 / CSS3 / JavaScript (ES6+)。
- **地圖引擎**: Leaflet.js 1.9.4。
- **資料格式**: 靜態 JSON (`food_data.json`)，包含店名、座標、地址、電話、連結等。
- **定位技術**: 瀏覽器 Geolocation API + 手動選點模式。
- **部署方式**: GitHub Pages / Netlify / Vercel 等靜態託管。

## 4. 資料處理 (ETL) 流程
1. **讀取資料**: 使用 Python (pandas) 讀取 `Alma_Food_Map_Final.xlsx`。
2. **座標轉換 (Geocoding)**: 使用 Geopy 或 Google Maps API 將地址轉換為緯度 (Latitude) 與經度 (Longitude)。
3. **資料清洗**: 修正營業時間格式、處理缺漏欄位。
4. **輸出封裝**: 產生最終的 `food_data.json` 供前端讀取。

## 5. 核心功能規格 (Core Functionality)
### 5.1 地圖交互
- **定位追蹤**: 顯示使用者當前位置（紅色標記）。
- **美食標記**: 以藍色標記 (📍) 顯示美食站點，點擊彈出資訊。
- **縮放層級控制**: 隨縮放比例動態調整標記顯示密度。

### 5.2 搜尋與導航
- **最近美食搜尋**: 依當前定位點找出最近的 5-10 處美食。
- **視覺導航線**: 從定位點至選取店家的直線引導（參照廁所快找之發光線條設計）。
- **食記連結**: 點擊後可跳轉至 Alma 部落格閱讀詳細內容。

### 5.3 定位狀態機 (同 TOILETS 規範)
1. **自動 GPS 模式**: 地圖隨移動實時更新。
2. **手動選點模式**: 使用者在地圖點擊自定義位置。
3. **手動鎖定模式**: 防止誤觸導致位置偏移。

## 6. 視覺與操作設計 (UI/UX Standards)
- **配色策略**: 採用高對比配色（例如：亮橘背景 #FF9F43 或深黃 #fbc531），強調「美食」聯想。
- **資訊面板**: 點擊標記時，從底部滑出詳細資訊面板（地址、電話、一鍵撥號、食記連結）。
- **長按功能**: 地址與電話支援長按複製。

## 7. 開發時程建議 (Roadmap)
- **Phase 1 (資料準備)**: 完成 Python ETL 腳本與座標轉換。
- **Phase 2 (地圖原型)**: 實作 Leaflet.js 基礎地圖與資料載入。
- **Phase 3 (功能開發)**: 加入定位追蹤、搜尋導航與 UI 優化。
- **Phase 4 (PWA 封裝)**: 加入 Service Worker 與 Manifest 達成離線可用。
